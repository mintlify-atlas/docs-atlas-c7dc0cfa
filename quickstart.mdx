This guide walks you through installing Gas Town, creating a workspace, adding a project, and running your first AI agent.

## Prerequisites

Before starting, ensure you have:
- **Go 1.25+** — [go.dev/dl](https://go.dev/dl/)
- **Git 2.25+**
- **Dolt 1.82.4+** — [dolthub/dolt](https://github.com/dolthub/dolt?tab=readme-ov-file#installation)
- **Beads (bd) 0.55.4+**
- **tmux 3.0+** (recommended)
- An AI coding agent CLI (Claude Code, Codex, Copilot, etc.)

See [Installation](/installation) for full setup instructions including installing each prerequisite.

## Run your first agent

<Steps>
  <Step title="Install Gas Town">
    <CodeGroup>
    ```bash Homebrew
    brew install gastown
    ```
    ```bash npm
    npm install -g @gastown/gt
    ```
    ```bash Go install
    go install github.com/steveyegge/gastown/cmd/gt@latest
    # Add to PATH if needed:
    export PATH="$PATH:$HOME/go/bin"
    ```
    </CodeGroup>

    Also install Beads:
    ```bash
    go install github.com/steveyegge/beads/cmd/bd@latest
    ```

    Verify both tools are installed:
    ```bash
    gt version
    bd version
    ```
  </Step>

  <Step title="Create your workspace">
    ```bash
    gt install ~/gt --git
    cd ~/gt
    ```

    This creates your Gas Town workspace (called a **Town**) at `~/gt/` with git initialized.

    Then start all services:
    ```bash
    gt up
    ```

    Run a health check to verify everything is working:
    ```bash
    gt doctor
    ```
  </Step>

  <Step title="Add a project">
    ```bash
    gt rig add myproject https://github.com/you/repo.git
    ```

    This creates a **Rig** — a project container that clones your repo and sets up agent workspaces. Replace the URL with your actual repository.

    <Tip>
      Each rig gets its own issue tracking (Beads), merge queue (Refinery), and agent monitor (Witness).
    </Tip>
  </Step>

  <Step title="Configure your agents">
    Check which AI agents are available:
    ```bash
    gt config agent list
    ```

    Gas Town ships with built-in support for `claude`, `gemini`, `codex`, `cursor`, `auggie`, `amp`, `opencode`, `copilot`, and more.

    Set your preferred default:
    ```bash
    gt config default-agent claude
    ```
  </Step>

  <Step title="Start the Mayor">
    The Mayor is your primary AI coordinator — it has full context about your workspace and manages agents on your behalf.

    ```bash
    gt mayor attach
    ```

    This opens a tmux session with your Mayor agent. Tell it what you want to build:

    ```
    Build a REST API endpoint for user authentication with JWT tokens.
    Create the necessary beads and assign them to agents.
    ```

    The Mayor will:
    - Break your goal into trackable work items (beads)
    - Create a convoy to group related tasks
    - Spawn polecat worker agents
    - Monitor progress and report back
  </Step>
</Steps>

## What just happened?

When you tell the Mayor what to build, Gas Town's orchestration layer kicks in:

1. **Mayor creates beads** — atomic work items in the Beads issue tracker
2. **Mayor creates a convoy** — groups related beads into a tracked delivery unit
3. **Mayor slingswork** — assigns beads to polecat worker agents via their Hooks
4. **Polecats execute** — each agent works in its own git worktree with persistent state
5. **Refinery merges** — completed work flows through the merge queue to main
6. **Mayor reports** — summarizes what was done and what's next

If an agent crashes or loses context, its work state is preserved in the git-backed Hook — it picks up exactly where it left off.

## Manual workflow (no Mayor)

For simpler tasks or when you want direct control:

```bash
# Create an issue in Beads
bd create --title="Fix auth bug" --type=bug --priority=1

# Create a convoy to track it
gt convoy create "Auth fixes" <bead-id>

# Assign to an agent
gt sling <bead-id> myproject

# Check progress
gt convoy list
gt agents
```

## Monitor your agents

```bash
# Real-time TUI activity feed
gt feed

# Web dashboard (default port 8080)
gt dashboard --open

# Check agent health
gt doctor
```
